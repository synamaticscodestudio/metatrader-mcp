Routing Configuration
Functions allow managing routing configurations, as well subscribe and unsubscribe from events associated with their change.
The following functions for managing routing are available:
| Function | Purpose |
| --- | --- |
| --- | --- |
| [RouteCreate](imtadminapi_routecreate.md "RouteCreate") | Create an object of a routing rule. |
| [RouteConditionCreate](imtadminapi_routeconditioncreate.md "RouteConditionCreate") | Create an object of an additional condition to apply a rule. |
| [RouteDealerCreate](imtadminapi_routedealercreate.md "RouteDealerCreate") | Create an object of a dealer configuration to whom requests under this rule will be sent. |
| [RouteSubscribe](imtadminapi_routesubscribe.md "RouteSubscribe") | Subscribe to events associated with the configuration of request routing. |
| [RouteUnsubscribe](imtadminapi_routeunsubscribe.md "RouteUnsubscribe") | Unsubscribe from events associated with the configuration of request routing. |
| [RouteUpdate](imtadminapi_routeupdate.md "RouteUpdate") | Add or update a routing rule. |
| [RouteUpdateBatch](imtadminapi_routeupdatebatch.md "RouteUpdateBatch") | Add or edit multiple routing rules. |
| [RouteDelete](imtadminapi_routedelete.md "RouteDelete") | Delete a routing rule by its name or index. |
| [RouteDeleteBatch](imtadminapi_routedeletebatch.md "RouteDeleteBatch") | Delete multiple routing routing rules. |
| [RouteShift](imtadminapi_routeshift.md "RouteShift") | Move a routing rule in the list. |
| [RouteTotal](imtadminapi_routetotal.md "RouteTotal") | The total number of routing rules available in the platform. |
| [RouteNext](imtadminapi_routenext.md "RouteNext") | Get a routing rule by the index. |
| [RouteGet](imtadminapi_routeget.md "RouteGet") | Get a routing rule by the name. |

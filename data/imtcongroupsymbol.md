Use of Default methods
Settings of trading instruments are defined by properties from [IMTConSymbol](imtconsymbol.md "IMTConSymbol"). They include basic symbol settings used for all clients by default. The platform allows overriding symbol settings for each client group. This can be done using the IMTConGroupSymbol class. In addition to Get and Set method used for working with overridden settings, the class also features Default methods, such as [IMTConGroupSymbol::TradeModeDefault](imtcongroupsymbol_trademodedefault.md "TradeModeDefault"). These methods can be used for the following purposes:
  * Checking if a certain symbol property has a default value for the group
  * Setting symbol properties to default for the group

Default methods do not return explicits value of basic symbol parameters and do not set default values.  
---  
In order to check whether a group's symbol property was inherited from a basic parameter, compare its value with the value of the corresponding Default method. For example:
//--- Creating an object of a client group's symbol settings IMTConGroupSymbol* group_symbol; ... //--- Checking if the default value is used if(group_symbol->TradeMode()==group_symbol->TradeModeDefault()) { //--- A default value is set } else { //--- An overridden value is set }  
---  
The set a symbol property to a default value, assign to it the value returned by the corresponding Default method. For example:
//--- Creating an object of a client group's symbol settings IMTConGroupSymbol* group_symbol; ... //--- Assigning a default value if(group_symbol->TradeMode(group_symbol->TradeModeDefault())!=MT_RET_OK) { //--- Setting a default value failed }  
---  
Example of how to request a list of available symbols for a group
This Server API example enables the receiving of a list of symbols available for the client group. The list is printed to the Journal in the order of symbol configurations in the group settings.
| //+---------------------------------+ // | Get available symbols in a group by client login | //+---------------------------------+ MTAPIRES CPluginInstance::GetGroupSymbols(UINT login) { //--- Check if(!m_api) { m_api->LoggerOut(MTLogErr, L"Failed to get group symbols by user, api interface not found"); return(MT_RET_ERROR); } //--- Create objects MTAPIRES retcode; IMTUser* user = m_api->UserCreate(); IMTConGroup* group = m_api->GroupCreate(); IMTConGroupSymbol* group_symbol = m_api->GroupSymbolCreate(); IMTConSymbol* symbol = m_api->SymbolCreate(); //--- get account information if(retcode=m_api->UserGet(login,user) != MT_RET_OK) { m_api->LoggerOut(MTLogErr, L"Failed to get user info"); return(retcode); } //--- Get the group of the client if(retcode = m_api->GroupGet(user->Group(), group) != MT_RET_OK) { m_api->LoggerOut(MTLogErr, L"Failed to get user group"); return(retcode); } //--- Get the list of symbols for the group for(int i = 0; i < group->SymbolTotal(); i++) { //--- Get entries one by one if(retcode = group->SymbolNext(i, group_symbol) !=MT_RET_OK) { m_api->LoggerOut(MTLogErr, L"Failed to get group symbol"); return(retcode); } //--- If the entry is a subdirectory, show all symbol from it, otherwise display the path to the symbol if(CMTStr::Find(group_symbol->Path(), L"*") >= 0) { for(int m = 0; m < m_api->SymbolTotal(); m++) { if(retcode = m_api->SymbolNext(m, symbol) != MT_RET_OK) { m_api->LoggerOut(MTLogOK, L"Failed to get symbol configuration"); return(retcode); } //--- remove '*' from the path CMTStr128 trim_path = group_symbol->Path(); trim_path.Trim(trim_path.Len()-1); //--- If path to the group symbol is the same as the beginning of the full symbol path, show the full path of the symbol if(CMTStr::Find(symbol->Path(), trim_path.Str()) == 0) { m_api->LoggerOut(MTLogOK, L"Symbol %s", symbol->Path()); retcode = MT_RET_OK; } } } else { m_api->LoggerOut(MTLogOK, L"Symbol %s",group_symbol->Path()); retcode = MT_RET_OK; } } //--- Release memory user->Release(); group->Release(); group_symbol->Release(); symbol->Release(); //--- All done return(retcode); } //+---------------------------------+ |

IMTGatewayAPI::DealerExecuteAsync
The platform notification on the order trade execution in the external system.
C++
MTAPIRES IMTGatewayAPI::DealerExecuteAsync( IMTExecution* execution // Trade execution object )  
---  
.NET
MTRetCode CIMTGatewayAPI.DealerExecuteAsync( CIMTExecution execution // Trade execution object )  
---  
Parameters
execution
[in] [Trade execution object](imtexecution.md "IMTExecution").
Return Value
If the command is successfully sent to the execution queue, the [MT_RET_OK](retcodes_successful.md "Successful completion") response code is returned. Otherwise, an error has occurred, which corresponds to the response code.
[MT_RET_OK_NONE](retcodes_successful.md "Successful completion") code is returned when the gateway is not able to determine the server to which the given trade execution should be sent based on the [IMTExecution type](imtexecution_action.md "Action") and the data passed in it. For example, IMTExecution of the TE_ORDER_NEW_REQUEST type does not include the order ticket on the MetaTrader 5 side.
The [MT_RET_ERR_CONNECTION](retcodes_common.md "Common errors") response code is returned if a trade server has not connected to anything yet since launching the gateway. After the first connection, the gateway is able to send executions using this method even if the connection to the trade server has been interrupted. In this case, the messages are sent during the next connection to the trade server.
Note
During the execution of this method the operation is carried out in the trading platform that corresponds to the executed one in the external environment.
The result of applying the [IMTExecution](imtexecution.md "IMTExecution") execution object to the server data base can be obtained in the [IMTGatewaySink::OnDealerAnswer](imtgatewaysink_ondealeranswer.md "OnDealerAnswer") method.
